1. General

What is hyperledger?

Hyperledger is an open source platform for creating private currencies. Anyone can create a ledger, and issue units to it as they wish. The system guarantees that transfers between accounts are always correct (no account can have a negative balance). While control over the issuance is centralised in one party, the system as a whole is decentralised. There are a number of nodes connected to each other in the consensus pool. Every request to one of them is broadcast to all the others. Once a critical number of nodes (more than two thirds) agree that the message is correct, the message is considered committed.

Is hyperledger open source?

Yes, both the hyperledger server and client are completely open source and can be found on github at:

hyperledger/hyperledger-server

hyperledger/hyperledger-cli

Both are released under the MIT License.

Is hyperledger an alt coin?

No, hyperledger is not a fork/clone/derivative of Bitcoin or its derivatives. It is an open source platform for the creation of currencies or assets.

Does hyperledger have its own currency?

No, hyperledger does not have its own designated currency or bridge currency. All currencies in the same hyperledger consensus pool are of equal stature.

Does hyperledger use proof of work or mining?

No, hyperledger does not use Proof of Work or mining, instead it relies on achieving consensus through a mechanism based on the Practical Byzantine Fault Tolerance algorithm.

Is hyperledger a company?

No, hyperledger is the name of the open source project that is not controlled by any single commercial entity.

How can I contribute?

Jump in on github or email us at contact@hyperledger.com

2. Consensus

What happens if one of the consensus nodes gets hacked?

If a node gets hacked there are only a few ways it can impact the system, but it can’t force good nodes to accept forged transfers. Generally, individual bad nodes will quickly be automatically identified and expelled from the consensus pool.

Is hyperledger vulnerable to a 51% attack?

hyperledger does not use Proof of Work mining and so is not vulnerable to the possibility of mining pools obtaining more than half the network. However, consensus in an asynchronous system with potentially malicious nodes requires greater than 2/3rds of the nodes to be honest but since bad nodes are automatically detected and expelled quickly, we don’t believe this is an issue.

What are the different pools?

testpoolis a totally open free for all where anyone can contribute a consensus node and do testing on their ledgers.

mainpoolis the primary global pool which will be supported by the wider community but there will be community determined restrictions on joining.

premiumpoolis a commercially supported pool with stricter requirements to join, including a membership subscription fee.

Can anyone start their own pool?

Yes. hyperledger is an open protocol with open source implementations and anyone is welcome to create their own pools, although they will lose some of the benefit of larger shared infrastructure.

Who will run the consensus nodes?

Consensus pools will probably have different requirements for node membership, but in the mainpool anyone with an interest in helping the platform and the capability to do so can run a consensus node.

Do I have to trust the consensus pool?

Consensus pools are made up of multiple entities, so collusion is highly unlikely, but you can also trace and verify all transactions yourself, if you so desire.

Could I run a hyperledger pool on a single server like a private database?

Yes, although you’d lose a lot of the benefits both in terms of resiliency and security.

What incentive is there to run a consensus node in the mainpool?

Anyone with an interest in seeing the pool be successful (e.g. ledger owners, exchanges, etc.) would have an incentive to run a node and its possible that some consensus pools could make it a requirement for running a ledger, for example.

What’s the requirement to run a consensus node?

This is still an open question. We want to keep it as accessible as possible but with some sort of nominal entry requirement to prevent control over a high percentage of the network. Different pools will almost definitely have different requirements.

How do you prevent SPAM/DDoS Attacks?

We will monitor the pool to see what kind of attacks the system comes under, but there are a number of methods than can be used to defend the system, including blacklisting clients and rate limiting. hyperledger is built on top of standard web technologies and so protecting the overall system is similar to any other distributed application.

How quickly is consensus achieved?

That can depend on the size of the pool and volume of transactions, but it normally takes less than a few seconds.

3. Ledgers

Does hyperledger use a Blockchain?

No, hyperledger uses a known set of nodes to agree on requests and sign the responses. The mechanism is based on the Practical Byzantine Fault Tolerance algorithm.

Is there only one big ledger?

No, a hyperledger pool contains multiple ledgers, which can be opened by anyone who then has control over the units in that ledger. Anyone can then open an account in that ledger (in the mainpool).

Can only one currency be issued on a single ledger by a single enti ty?

Yes, each ledger can only contain one asset type and all accounts on this ledger can only contain that one asset type. Each ledger can only have one primary account, although theoretically you could allow multiple entities to have access to that account.

Can anyone start a ledger?

Yes. There are no restrictions on creating ledgers in the mainpool.

What if there are two ledgers with the same name in different pools?

This is an open question up for discussion.

Does a ledger have to be used for a currency?

No, a ledger can be used to record anything that has a fixed number of units, for example shares in a company, vouchers, phone credit, or equity in a property.

Are currencies on hyperledger fixed issuance?

The issuing rules of the currency are determined by the entity that opens the ledger.

What’s to stop a consensus node/hacker changing balances in a ledger?

The balances on accounts in the system are computed from a list of transfers, which are signed by the account private key holder. If the attacker cannot get hold of this, they cannot forge transfers in the system.

4. Transactions

How many transactions per second are possible?

We haven’t fully load-tested the system yet, but we believe it’s capable of a few thousand transactions per second.

Are transactions reversible?

No, at the moment all transactions are final, but it is possible that new pools in the future could enable reversible transactions.

Are transactions private?

At the moment all transactions are published openly, however, in the future there may be consensus pools that do keep transactions private.

Are transactions anonymous?

Transactions are pseudonymous, as they are in Bitcoin.

Are there any transaction fees?

There are no transaction fees built into the system, but consensus pools may have their own rules.

How do you prevent double spending?

The consensus pool sorts all transfers into an agreed order and completes transfers as long as there’s an available balance. Due to the ordering of transactions, any attempt to double spend will result in the second transaction failing.

How long before a transaction is complete?

That can depend on the state of the system but a transaction should complete in a few seconds at most.

How can a transaction be verified?

All transfers are returned with a list of the signatures from the consensus nodes. The nodes publish their public key so the signatures can be checked.

Do I need to download a copy of the ledger to verify payments?

No, the transactions are verified on the consensus pool nodes, so once you receive notified that it is successful you don’t need to verify it against a local copy.

5. How does it compare to…

Ripple

hyperledger can be seen as an alternative or complementary system to Ripple, depending on what you are hoping to achieve.

Ripple is centered around trusted lines of credit between entities and finding least-cost paths through that trust graph to settle transactions, possible in different currencies. hyperledger’s aim is to allow private entities to easily create a new currency and have precise control over the issuance of units of that money.

hyperledger also has a distinct notion of consensus pools, which, while adhering to the same protocol, may have different rules about membership and privacy.

hyperledger does not have a special currency to use the system.

On a more technical level, hyperledger is aiming for a slightly different consensus scheme based on PBFT and ordering each request to the system, rather than Ripple’s method of ‘Continuous Ledger Close’. The 2 systems are closely related, however.

Note there’s a slight difference in terminology; Ripple uses ‘ledger’ to refer to a grouped set of requests to the system, whereas in hyperledger it refers to the set of accounts that use the same currency, and the records of issuances and transfers.
